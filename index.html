html

Html is the backbone of a web design. It contains all the content, but only in its simplist form. Think of them like your ingredients.

HTML Regions

All html documents generally follow a similar format. The head tag is always first, followed by the body, a header, perhaps a nav, then the main section, and finally a footer.

<head>

The head tag in HTML contains information about the web page that is not displayed on the actual page itself. This includes things like the title of the page, links to CSS stylesheets, and metadata such as keywords and descriptions that can help search engines index the page properly.

<body>

This tag is where the main content of a website will live. Because all items within the body tag are expected to be visible to the user, a website's header and navigation will be written within the body tag.

<header>

The header will usually contain a website's logo and manu navigation page. Not to be confused with the <head> tag that holds information about the webpage that is not displayed to the user directly.

<nav>

The menu or "navigation" links to a website live within this tag. The nav tag can usually be found either within a header tag or on its own.

<main>

This tag denotes the start of a website's "main" content. This can include blog posts, pictures, and other content that will appeal to a user.

<footer>

This is the last item in an html file. The footer can contain as simple or complex as the designer wishes. Some simply have a copyright message, whike others may choose to include a call to action such as a email list sign up or even an additional navigation menu.

Common HTML Tags and Elements

I like to divide the html elements into 2 types:

Blocks

These elements help hold and organize the content of the website rather than generating the content. These are like your plates and bowls.

<div>

The most common and confusing block element. You could technically get away with using divs for everything but it is not good practise. Because a div can be as large or small as you want, it is reccomended and add classes or ids to them to differenciate between their function.

<section>

The largest of the block tags. A section is usually definted whenever a heading tag is used, such is h1, h2, or h3. It is only reccomended to use a section tag if the topic of your content changes.
